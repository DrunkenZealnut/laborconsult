# Plan: 챗봇 메인페이지 제작

> Feature: `chatbot-main-page`
> Created: 2026-03-18
> Level: Dynamic

## Executive Summary

| Perspective | Description |
|-------------|-------------|
| **Problem** | 현재 메인페이지가 채팅창 단일 기능으로, 신규 사용자가 어떤 질문이 가능한지 모르고 계산기 기능 접근이 어려움 |
| **Solution** | FAQ(카테고리별), 임금 계산기 바로가기, 질문게시판을 포함한 다기능 메인페이지로 리디자인 |

| **Core Value** | 사용자 진입 장벽 감소 + 서비스 활용도 극대화 → 청년노동자인권센터 미션 달성 |

## 1. Background & Problem

### 1.1 현재 상태
- `public/index.html`: 채팅창 단일 인터페이스 (header + chat + input)
- `public/calculators.html`: 25개 임금계산기 흐름도 페이지 (별도 페이지)
- `public/admin.html`: 관리자 대시보드
- 신규 사용자는 "무엇을 질문할 수 있는지" 파악이 어려움
- 계산기 기능 접근은 header의 작은 링크("📊 흐름도")에 의존

### 1.2 문제점
1. **탐색성 부족**: 채팅 입력창만 보이므로 가능한 질문 범위를 모름
2. **계산기 접근성 부족**: 25개 계산기가 있지만 바로가기가 없음
3. **정보 부족**: 조직 정보(청년노동자인권센터)가 노출되지 않음
4. **커뮤니티 부재**: 다른 사용자의 질문을 볼 수 없음

## 2. Goal

현재 채팅 전용 페이지를 **다기능 메인페이지**로 리디자인하여:

- 신규 사용자가 바로 원하는 정보를 찾을 수 있게 함
- 임금 계산기에 빠르게 접근 가능
- 커뮤니티 질문게시판으로 참여 유도
- 조직 정보 노출로 신뢰도 확보

## 3. Scope

### 3.1 In-Scope

#### SEC-01: 챗봇 채팅창 (Hero Section)
- 기존 채팅 기능 유지 (SSE 스트리밍, 파일 첨부, 마크다운 렌더링)
- 메인페이지 상단에 배치, 간결한 인사말 + 입력창
- 기존 `index.html`의 모든 JS 로직 보존

#### SEC-02: 많이 하는 질문 (FAQ 카테고리별)
- 카테고리 분류:
  - **임금/수당**: 주휴수당, 연장수당, 야간수당, 최저임금
  - **퇴직/해고**: 퇴직금, 해고수당, 해고예고수당
  - **근로계약**: 근로계약서, 수습기간, 계약직
  - **4대보험**: 건강보험, 국민연금, 고용보험, 산재보험
  - **직장 내 괴롭힘**: 판단 기준, 신고 방법, 대응 절차
  - **출산/육아**: 출산휴가, 육아휴직, 급여 계산
- 각 카테고리별 대표 질문 3~5개 표시
- 질문 클릭 시 챗봇 입력창에 자동 입력 → 즉시 전송

#### SEC-03: 임금 계산기 바로가기
- **표현 제안**: "임금 계산기" (기존 `calculators.html` 제목 "임금계산기 흐름도"와 일관)
  - 대안: "급여 계산 도구", "노동법 계산기"
- 주요 계산기 카드형 바로가기 (6~8개):
  - 주휴수당 계산기
  - 퇴직금 계산기
  - 연장/야간/휴일수당 계산기
  - 최저임금 검증
  - 실업급여 계산기
  - 4대보험료 계산기
  - 연차수당 계산기
  - 해고예고수당 계산기
- 카드 클릭 시 해당 계산 질문 예시를 채팅에 자동 입력
- "전체 계산기 보기" → `calculators.html` 링크

#### SEC-04: 질문게시판
- 다른 사용자들의 최근 질문/답변 목록 표시
- Supabase `conversations` 테이블에서 최근 공개 질문 조회
- 개인정보 비식별화 처리 (이름, 회사명 마스킹)
- 질문 클릭 시 해당 답변 펼침 또는 유사 질문으로 채팅 시작
- 페이지네이션 (10건씩)

#### SEC-05: Footer
- **조직 정보**:
  - 명칭: 청년노동자인권센터
  - 주소: 서울시 종로구 성균관로12 5층
  - 이메일: admin@younglabor.kr
- 면책 고지 (현재 하단 disclaimer 통합)
- Copyright 표시
- 주요 링크 (개인정보처리방침, 이용약관 - 플레이스홀더)

### 3.2 Out-of-Scope
- 회원가입/로그인 기능 (향후 별도 feature)
- 질문게시판 글쓰기 기능 (현 단계에서는 읽기 전용)
- 다국어 지원
- 다크모드 (향후 개선)

## 4. Technical Approach

### 4.1 Architecture
- **Single Page**: `public/index.html` 하나의 파일로 유지 (Vercel + GitHub Pages 배포 호환)
- **레이아웃**: 스크롤 기반 섹션 구조
  ```
  ┌─────────────────────────────┐
  │         Header              │
  ├─────────────────────────────┤
  │    SEC-01: 챗봇 채팅창       │  ← Hero, 화면 높이의 60-70%
  ├─────────────────────────────┤
  │    SEC-02: 많이 하는 질문     │  ← 카테고리 탭 + 질문 리스트
  ├─────────────────────────────┤
  │    SEC-03: 임금 계산기        │  ← 카드 그리드 (2x4 or 3x3)
  ├─────────────────────────────┤
  │    SEC-04: 질문게시판         │  ← 리스트형, 펼침/접기
  ├─────────────────────────────┤
  │    SEC-05: Footer            │  ← 조직 정보 + 면책
  └─────────────────────────────┘
  ```

### 4.2 기술 스택
- **Frontend**: Vanilla HTML/CSS/JS (기존 유지, 외부 프레임워크 없음)
- **Backend**: 기존 FastAPI 활용 (질문게시판 API 추가 필요)
- **DB**: Supabase (기존 conversations 테이블 활용)
- **배포**: Vercel (API + static) + GitHub Pages (static fallback)

### 4.3 API 추가 필요
- `GET /api/faq/popular` — 인기 질문 목록 (카테고리별, 캐싱)
- `GET /api/board/recent` — 최근 공개 질문/답변 (비식별화, 페이지네이션)

### 4.4 성능 고려
- FAQ 데이터: 하드코딩 또는 빌드 시 생성 (API 호출 최소화)
- 질문게시판: lazy loading (스크롤 도달 시 로드)
- 채팅 영역: 기존 SSE 로직 100% 보존
- 모바일 우선 반응형 디자인

## 5. Risk & Mitigation

| Risk | Impact | Probability | Mitigation |
|------|--------|-------------|------------|
| 채팅 기능 회귀 | High | Medium | 기존 JS 로직 분리 보존, E2E 테스트 |
| 질문게시판 개인정보 노출 | High | Low | 서버사이드 비식별화, 민감 키워드 필터링 |
| 페이지 로드 성능 저하 | Medium | Low | 섹션별 lazy loading, 이미지 최적화 |
| 모바일 레이아웃 깨짐 | Medium | Medium | 모바일 우선 설계, 실기기 테스트 |

## 6. Success Criteria

| Metric | Target |
|--------|--------|
| 기존 채팅 기능 정상 동작 | 100% (회귀 없음) |
| FAQ 질문 클릭 → 챗봇 응답 | 2초 이내 |
| 계산기 바로가기 → 계산 시작 | 2클릭 이내 |
| 모바일 반응형 | 320px ~ 1440px 정상 렌더링 |
| Lighthouse 성능 점수 | 90+ |
| 질문게시판 로드 | 1초 이내 (10건 기준) |

## 7. Implementation Priority

1. **P0**: SEC-01 (채팅창 보존) + SEC-05 (Footer) — 핵심 + 기본 정보
2. **P1**: SEC-02 (FAQ) + SEC-03 (임금 계산기) — 탐색성 + 접근성
3. **P2**: SEC-04 (질문게시판) — API 추가 필요, 복잡도 높음

- 기존 `public/calculators.html` 계산기 흐름도
