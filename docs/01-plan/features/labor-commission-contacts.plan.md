## 1. Problem Statement

### 현재 상태 (AS-IS)

- 챗봇 답변 말미에 "고용노동부(1350) 또는 노무사에게 문의하세요"만 반복
- 노동위원회 관련 정보가 전혀 없음 — 부당해고 구제신청, 차별시정, 노동쟁의 조정 등 노동위원회 소관 사안도 "1350"으로 안내
- 사용자가 관할 노동위원회를 직접 검색해야 하는 불편

### 목표 상태 (TO-BE)

- 전국 14개 노동위원회 연락처 데이터 내장 (중앙 1 + 지방 13)
- 해고·차별시정·노동쟁의 등 노동위원회 소관 사안 질문 시 관할 노동위원회 자동 안내
- 사용자가 지역을 언급하면 해당 지역 관할 노동위원회 연락처 제공
- 지역 미언급 시 전국 노동위원회 목록 요약 제공

### 근거

- nlrc.go.kr (중앙노동위원회) 공식 사이트에서 14개 기관 연락처 확인 완료
- 부당해고 구제신청은 노동위원회가 유일한 접수 기관 — 고용노동부(1350)와는 별개

## 3. Scope

### In-Scope

1. **노동위원회 데이터 모듈** (`labor_offices.py`): 14개 기관 연락처 + 관할구역 매핑
2. **관할구역 자동 매칭**: 사용자 질문에서 지역명 추출 → 해당 지방노동위원회 안내
3. **챗봇 시스템 프롬프트 업데이트**: 노동위원회 소관 사안 판단 시 연락처 포함 지시
4. **파이프라인 통합**: 해고·차별·노동쟁의 관련 답변에 관할 노동위원회 정보 자동 삽입

### Out-of-Scope

- 고용노동부 지방관서(근로감독관) 연락처 (별도 feature)
- 노동위원회 온라인 구제신청 안내 (별도 feature)
- 실시간 노동위원회 운영시간 조회
- 지도 기반 위치 안내 UI

## 4. Technical Approach

### 4.1 데이터 구조

```python
@dataclass
class LaborCommission:
    name: str           # 기관명
    phone: str          # 대표전화
    fax: str            # 팩스
    address: str        # 주소
    jurisdiction: list[str]  # 관할 지역 키워드 (서울, 부산, 대구·경북 등)
```

### 4.2 관할구역 매핑 전략

지역명 키워드 → 관할 노동위원회 매핑:
- "서울", "강남", "마포" → 서울지방노동위원회
- "부산", "해운대" → 부산지방노동위원회
- "대구", "경북", "포항" → 경북지방노동위원회
- "대전", "충남", "세종" → 충남지방노동위원회
- "광주", "전남", "목포" → 전남지방노동위원회

### 4.3 트리거 조건

노동위원회 연락처 제공이 필요한 사안:
- 부당해고 구제신청
- 부당노동행위 구제신청
- 차별시정 신청
- 노동쟁의 조정·중재
- 교섭대표 결정
