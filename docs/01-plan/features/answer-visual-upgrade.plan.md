## 1. 배경 및 목적

### 1.1 현재 상태

현재 `public/index.html`의 `md()` 함수가 markdown을 HTML로 변환하지만, 결과물은 **기본적인 텍스트 스타일링** 수준:

- `##` → 단순 heading (16px/15px, margin만 조정)
- `>` → 회색 왼쪽 border의 blockquote
- `|table|` → 기본 border table
- `**bold**` → font-weight: 600
- 리스트 → 기본 ul/ol

**문제점:**
1. 법적 근거, 주의사항, 계산 결과, 절차 안내가 모두 같은 스타일로 표시됨
2. 긴 답변에서 핵심 정보를 빠르게 찾기 어려움
3. 계산 결과가 텍스트로만 표현되어 직관성이 떨어짐
4. 단계별 절차가 단순 번호 목록이라 진행 흐름을 파악하기 어려움
5. 학습자료나 교과서와 비교하면 정보 전달 효과가 현저히 낮음

### 1.2 목표 상태

답변이 **노동법 학습자료** 수준의 시각적 품질을 가지도록:

1. **콜아웃 박스**: 법적 근거(파랑), 주의사항(주황), 팁(초록), 중요(빨강) 등 색상 코딩
2. **계산 결과 카드**: 항목별 금액이 시각적으로 구분되는 하이라이트 카드
3. **단계 표시기**: 절차 안내가 시각적 스텝 인디케이터로 표현
4. **요약 배지**: 핵심 결론이 눈에 띄는 요약 박스로 표시
5. **향상된 테이블**: 교대 행 색상, 헤더 색상 강화, 합계 행 하이라이트
6. **모바일 퍼스트**: 모든 컴포넌트가 모바일에서 먼저 최적화

## 2. 구현 전략

### 2.1 접근 방식: CSS 리치 컴포넌트

**핵심 원칙**: 백엔드 변경 최소화. LLM이 이미 생성하는 markdown 패턴을 프론트엔드에서 시각적 컴포넌트로 자동 변환.

```
LLM 답변 (markdown) → md() 렌더러 확장 → CSS 리치 컴포넌트 → 학습자료 수준 UI
```

### 2.2 변환 규칙 (markdown 패턴 → 리치 컴포넌트)

| Markdown 패턴 | 리치 컴포넌트 | 시각적 효과 |
|---|---|---|
| `> **법적 근거**` 또는 `> 📘` | 법적 근거 콜아웃 (파랑) | 파란 좌측 보더 + 파란 배경 |
| `> **주의사항**` 또는 `> ⚠️` | 경고 콜아웃 (주황) | 주황 좌측 보더 + 주황 배경 |
| `> **중요**` 또는 `> 🚨` | 중요 콜아웃 (빨강) | 빨간 좌측 보더 + 빨간 배경 |
| `> **참고**` 또는 `> 💡` | 참고 콜아웃 (초록) | 초록 좌측 보더 + 초록 배경 |
| `## 📊 계산 결과` | 계산 결과 카드 | 그림자 + 배경색 + 하이라이트 |
| `## ⚖️` 또는 `## 핵심 답변` | 요약 배지 | 강조 배경 + 큰 텍스트 |
| 테이블 with 금액 | 향상된 테이블 | 교대 행색 + 합계 행 강조 |
| 번호 목록 (절차) | 스텝 인디케이터 | 원형 번호 + 연결선 |
| `---` (구분선) | 섹션 구분자 | 여백 확대 + 미세한 그라디언트 |

### 2.3 시스템 프롬프트 조정

LLM이 시각적 컴포넌트에 맞는 markdown을 생성하도록 프롬프트 규칙 업데이트:

```
현재: "핵심 금액·결론은 **볼드**로, 법조문명은 **볼드**로 표시하세요"
변경: "법적 근거는 '> 📘 **법적 근거**' 형식으로, 주의사항은 '> ⚠️ **주의사항**' 형식으로 작성하세요"
```

## 3. 상세 구현 항목

### 3.1 콜아웃 박스 시스템 (최우선)

blockquote 내 키워드/이모지를 감지하여 색상 코딩된 콜아웃으로 변환.

**4가지 타입:**

| 타입 | 트리거 키워드 | 색상 | 용도 |
|------|---|---|---|
| `legal` | 법적 근거, 관련 법조문, 📘 | `#dbeafe` / `#2563eb` | 법조문 인용, 판례 요지 |
| `warning` | 주의사항, 주의, ⚠️ | `#fef3c7` / `#d97706` | 적용 제외, 예외 사항 |
| `danger` | 중요, 필수, 🚨 | `#fee2e2` / `#dc2626` | 절대 규칙, 위반 시 불이익 |
| `tip` | 참고, 팁, 💡, 알아두세요 | `#dcfce7` / `#16a34a` | 실무 팁, 추가 정보 |

**구현 위치**: `md()` 함수 내 blockquote 처리 로직 확장

### 3.2 계산 결과 카드

계산기 결과 섹션을 시각적으로 돋보이는 카드로 표현.

**구성 요소:**
- 카드 헤더 (아이콘 + 제목)
- 핵심 금액 대형 표시 (월 총수령액 등)
- 항목별 내역 테이블 (향상된 스타일)
- 계산식 영역 (코드 블록 스타일 대신 수식 카드)
- 법적 근거 연결 (콜아웃 박스로)

**구현 방식**: `## 📊 계산 결과` 또는 `## 계산 결과` heading 이후 콘텐츠를 카드 wrapper로 감싸기

### 3.3 향상된 테이블

| 개선 항목 | 현재 | 개선 후 |
|---|---|---|
| 헤더 행 | `--bg` (연회색) | 진한 `--primary` 배경 + 흰 텍스트 |
| 데이터 행 | 동일 배경 | 교대 행 색상 (zebra striping) |
| 합계 행 | 구분 없음 | 볼드 + 상단 border 강조 |
| 금액 컬럼 | 좌측 정렬 | 우측 정렬 + 고정폭 숫자 폰트 |
| 모바일 | overflow-x scroll | 카드형 변환 (선택적) |

### 3.4 단계 표시기 (Step Indicator)

절차 안내용 번호 목록을 시각적 스텝 인디케이터로 변환.

**트리거 조건**: `## 절차`, `## 대응 절차`, `## 신청 방법` 등의 heading 아래에 있는 `ol` 목록

**시각적 구현:**
```
 ① ─── 관할 고용노동부 확인
 │
 ② ─── 진정서 작성 (서식 다운로드)
 │
 ③ ─── 고용노동부 제출 (온라인/방문)
 │
 ④ ─── 조사 및 시정지시
```

### 3.5 요약 배지 / 핵심 답변 박스

답변 첫 부분의 핵심 결론을 눈에 띄는 박스로 표시.

**트리거**: `## 핵심 답변`, `## ⚖️`, `## 결론` heading

**시각적 효과**: 약간 큰 폰트 + 좌측 굵은 primary 보더 + 연한 primary 배경

### 3.6 구분선 개선

`---` (hr) 태그를 더 시각적으로 의미 있는 섹션 구분자로 변환.

**현재**: 기본 `<hr>` (회색 선)
**개선**: 위아래 여백 확대 + 그라디언트 또는 점선 스타일 + 섹션 전환 느낌

### 3.7 면책 고지 스타일링

답변 끝의 면책 고지를 별도 스타일로 처리.

**트리거**: "본 답변은 참고용이며" 또는 "법적 효력이 없습니다" 패턴
**스타일**: 작은 폰트 + muted 색상 + 상단 구분선 + 약간의 패딩

## 6. 비기능 요구사항

### 6.1 성능
- CSS 추가로 인한 initial load 증가: **< 5KB**
- md() 함수 실행 시간 증가: **< 10ms** (패턴 매칭 추가)
- 스트리밍 중 레이아웃 시프트 최소화

### 6.2 모바일 퍼스트
- 콜아웃 박스: 전체 너비 사용, 패딩 축소
- 테이블: 기존 가로 스크롤 유지 + 교대 행 색상
- 스텝 인디케이터: 세로 레이아웃 (모바일 기본)
- 터치 타겟: 기존 44px 최소 유지

### 6.3 접근성
- 콜아웃 박스: `role="note"` 또는 적절한 aria 속성
- 색상만으로 정보 전달하지 않음 (아이콘 + 텍스트 병행)
- 고대비 모드 호환

### 6.4 하위 호환성
- 기존 markdown이 깨지지 않아야 함
- 콜아웃 트리거 패턴에 매칭되지 않는 blockquote는 현재 스타일 유지
- 기존 SSE 이벤트 구조 변경 없음

| CSS 파일 크기 증가로 초기 로딩 지연 | Low | 인라인 CSS이므로 추가 요청 없음, < 5KB |

## 9. 제외 범위 (YAGNI)

- Chart.js 등 외부 차트 라이브러리 도입 (현 단계에서는 CSS만으로 충분)
- 새로운 SSE 이벤트 타입 추가 (CSS 리치 컴포넌트 방식으로 해결)
- 다크 모드 (현재 프로젝트에 없음, 별도 feature로)
- 애니메이션/트랜지션 (성능 우선, 필요 시 후속으로)
- PDF 내보내기 시 스타일 반영 (현재 markdown 내보내기만 지원)
