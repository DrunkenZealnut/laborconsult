## 2. 기능 요구사항 (TO-BE)

### 2.1 계산 대상 급여 항목 (6개)

| # | 급여 항목 | 계산 공식 | 법적 근거 |
|---|----------|----------|----------|
| G1 | 휴업급여 | 평균임금 × 70% × 요양일수 | 산재보험법 제52조 |
| G2 | 상병보상연금 | 평균임금 × 등급별 연금일수 (329/291/257일) | 산재보험법 제66조 |
| G3 | 장해급여(연금) | 평균임금 × 등급별 연금일수 (1~7급) | 산재보험법 제57조 |
| G4 | 장해급여(일시금) | 평균임금 × 등급별 일시금일수 (1~14급) | 산재보험법 제57조 |
| G5 | 유족급여(연금) | 평균임금 × 365 × (47% + 유족수×5%) | 산재보험법 제62조 |
| G6 | 유족급여(일시금) | 평균임금 × 1,300일 | 산재보험법 제62조 |
| G7 | 장례비 | 평균임금 × 120일 (최고/최저액 적용) | 산재보험법 제71조 |

### 2.3 휴업급여 최저보상기준 적용 로직

```
1단계: avg_daily × 0.70 계산
2단계: 결과가 최저보상기준 × 80% 이하인 경우
  → avg_daily × 0.90 적용
3단계: avg_daily × 0.90이 최저보상기준 × 80% 초과인 경우
  → 최저보상기준 × 80% 적용
4단계: 위 금액이 최저임금 미만인 경우
  → 최저임금 적용
```

## 3. 입력 데이터 모델 확장

### 3.1 WageInput 신규 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `accident_date` | str | 산재 발생일 (YYYY-MM-DD) |
| `disability_grade` | int | 장해등급 (1~14, 0=해당없음) |
| `disability_pension` | bool | 장해급여 연금 선택 여부 (True=연금, False=일시금) |
| `severe_illness_grade` | int | 중증요양상태 등급 (1~3, 0=해당없음) |
| `num_survivors` | int | 유족수 (0~4+) |
| `survivor_pension` | bool | 유족급여 연금 선택 여부 |
| `sick_leave_days` | int | 요양(휴업) 일수 |
| `is_deceased` | bool | 사망 여부 |

### 3.2 평균임금 연동

- `average_wage` target이 함께 실행되면 `AverageWageResult.avg_daily_wage` 사용
- 독립 실행 시 `WageInput.daily_ordinary_wage` 또는 직접 입력된 평균임금 사용

## 6. 위험 요소

| 위험 | 영향 | 대응 |
|------|------|------|
| 평균임금 계산 모듈 부재 시 | 산재보상금 계산 불가 | average_wage target 자동 포함 또는 직접 입력 허용 |

| 유족수 4명 이상 상한 | 67% 초과 불가 | min(67, 47+5*n) 처리 |
| 최저보상기준 연도별 변동 | 연도별 금액 차이 | constants.py에 연도별 관리 |

## 7. 연동 포인트

- **평균임금 계산기** (`average_wage.py`): 3개월 임금 기반 1일 평균임금 → 산재보상금 기초
- **통상임금 계산기** (`ordinary_wage.py`): 평균임금 < 통상임금 시 대체
- **보험 계산기** (`insurance.py`): 산재보험료율 참조
- **chatbot 연동**: CALC_TYPE_MAP에 "산재보상", "휴업급여", "장해급여", "유족급여" 키워드 등록
